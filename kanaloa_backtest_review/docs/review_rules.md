# バックテスト検証の評価ルール

## 分類

### 1. 妥当なA判定

- A判定が出て、結果も好走
- 反応要素が再現性あり
- オッズや展開も合っていた
- ただし予想実績としては扱わない

### 2. 過剰評価A判定

- A判定が出たが凡走
- 血統や過去実績だけを強く見すぎた
- 馬場、展開、ペース、位置取り、距離適性の逆風を見落とした
- 投資額を下げるべきだった可能性

### 3. 見落とし好走

- B/C判定だったが好走
- 評価できていないタグや条件があった可能性
- 新しい加点候補を探す

### 4. 正しい見送り

- C判定で凡走
- 買わない判断が妥当

### 5. 度外視可能不発

- 度外視可能タグは妥当だったが、条件好転が足りなかった
- 見直しと買いを混同していた可能性

### 6. 次走狙い候補

- 結果は悪いが、内容が悪くない
- 外回し、前壁、馬場不向き、展開不向き、ペース負荷などがある
- 次走条件好転で見直し

## 必ず見る観点

- レース名
- 馬名
- AI判定
- 単勝期待値
- 複勝期待値
- 推奨投資額
- 人気
- オッズ
- 着順
- 着差
- 通過順
- 上がり
- 馬場
- 展開
- 距離変更
- 芝/ダート替わり
- 血統
- 馬体重
- 騎手
- 適用要素
- 備考タグ
- 週末馬場バイアスがある場合はその影響
