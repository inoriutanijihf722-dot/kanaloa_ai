# Codex用プロンプト: Eカナロア診断入力補助ワークフロー

## 使い方

このプロンプトは、添付スクショから人気ランクEカナロア候補を抽出し、Kanaloa AI Pro Ver13.5 の「人気ランクEカナロア抽出β」と「診断入力補助β」を安全に使うためにCodexへ渡す。

以下のプロンプト本文を、スクショと一緒にCodexへ渡す。

---

## プロンプト本文

Kanaloa AI Pro Ver13.5 のEカナロア診断入力補助βを使って、添付スクショの馬を【Eカナロア母集団】候補として確認し、診断入力フォームへ下書き反映できるか確認してください。

作業場所:

```text
/Users/fukushigehiroshi/Desktop/kanaroa_ai
```

対象UI:

```text
AI診断・入力
↓
人気ランクEカナロア抽出β
↓
OCR結果貼り付け欄
↓
確認済みチェック
↓
診断入力用サマリー
↓
馬1フォームへ下書き反映
```

## 目的

人気ランクEで、父または母父にロードカナロア/カナロアが確認できる馬を【Eカナロア母集団】候補として扱う。

これは母集団抽出と診断入力補助だけの作業です。買い推奨、A/B/C判定、期待値計算、推奨額計算、保存、CSV追記、【ディープバリュー候補】付与は行わない。

## 入力するスクショ情報

スクショから分かる範囲で、OCR結果貼り付け欄に以下の形式で入れる。

```text
東京4R
レース名 3歳新馬
馬番 9
馬名 ジーディーミュウ
性齢 牝3
人気ランク E
単勝オッズ 234.5
騎手 上里直汰
厩舎 柄崎将寿
馬主 田畑 利彦
生産者 高昭牧場
父 ノーブルミッション
母父 ロードカナロア
母母父 メジロルルド
備考 三代血統表より母父ロードカナロア。人気ランクE。単勝234.5。
```

別の馬を扱う場合も、馬名や数値をハードコードせず、スクショから読める値に置き換える。

## Eカナロア抽出βの確認

以下を確認する。

- 人気ランクがEである
- 父または母父にロードカナロア/カナロアがある
- `候補判定` が `✅ 【Eカナロア母集団】候補` になる
- `deep_value_tag` は空のまま
- 【ディープバリュー候補】は自動付与されない

## 確認済みチェック

内容を目視確認した候補だけ、確認済みにチェックする。

- 確認済み0頭の場合、馬1フォームへの下書き反映ボタンは出ない
- 確認済み1頭の場合だけ、馬1フォームへの下書き反映ボタンが出る
- 確認済み2頭以上の場合、誤反映防止のためボタンは出ない

## 診断入力用サマリー

確認済み候補では、以下の診断入力用サマリーを確認する。

```text
馬名
馬主
タイプ
性別
騎手
厩舎
母父(父)
人気
単勝
備考
```

ジーディーミュウの期待例:

```text
馬名: ジーディーミュウ
馬主: 田畑 利彦
タイプ: 母父カナロア
性別: 牝
騎手: 上里直汰
厩舎: 柄崎将寿
母父(父): ロードカナロア
人気: E
単勝: 234.5
備考: 【Eカナロア母集団】候補 父:ノーブルミッション / 母父:ロードカナロア / タイプ:母父カナロア / 人気:E / 単勝:234.5
```

## 馬1フォームへ下書き反映

確認済み候補が1頭だけの場合に限り、「馬1フォームへ下書き反映」を押してよい。

反映後は、馬1フォームを必ず目視確認する。

確認する項目:

```text
馬名 -> n_0
馬主 -> ow_0
タイプ -> t_0
性別 -> s_0
騎手 -> j_0
厩舎 -> tr_0
母父(父) -> mf_0
人気 -> r_0
単勝 -> o_0
```

空欄・未確定値・Noneは反映しない。既存入力を空欄で上書きしない。

## 絶対にしないこと

- AI診断実行ボタンを押さない
- 保存しない
- CSV追記しない
- `kanaloa_investment_log.csv` を変更しない
- `scoring.py` を変更しない
- A/B/C判定ロジックを変更しない
- 期待値計算を変更しない
- 推奨額ロジックを変更しない
- `uv.lock` を変更しない
- `.vscode/` に触らない
- 【ディープバリュー候補】を自動付与しない

## 確認結果として報告すること

- OCR貼り付け内容
- 【Eカナロア母集団】候補になったか
- カナロア該当タイプ
- 診断入力用サマリー
- 馬1フォームへ反映した場合は、反映後の各フォーム値
- AI診断実行・保存・CSV追記をしていないこと
- `git status --short`
